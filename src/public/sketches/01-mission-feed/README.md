## Variante 1 : Mission Feed

### Design stance
Le dashboard traditionnel disparaît. L'utilisateur arrive directement sur **ce qui compte** : les missions disponibles + ses réservations en cours. C'est le modèle "feed" (Uber, Airbnb, Leboncoin).

### Key choices
- **Layout** : une colonne, scroll vertical fluide
- **Suppression** : pas de stats, pas de progression, pas de badges, pas d'alertes
- **Info essentielle** : solde TIME en haut (toujours visible)
- **3 actions** : Explorer / Proposer / Mes résas — les seuls vrais besoins
- **Bottom nav** mobile : 5 onglets permanents (Accueil, Missions, Résas, Wallet, Profil)

### Trade-offs
- **Fort** : ultra-rapide à comprendre, focus sur l'action immédiate, zéro bruit
- **Faible** : pas de gamification visible, pas de vue d'ensemble "tableau de bord"
- **Risque** : l'utilisateur "passif" pourrait ne pas savoir quoi faire — compensé par le time-bar visible

### Best for
- Nouveaux utilisateurs qui veulent juste trouver ou proposer une mission
- Le directeur de la Fondation qui veut comprendre le produit en 5 secondes
